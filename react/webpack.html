webpack 개념

webpack는 웹 애플리케이션을 위한 정적 모듈 번들러(Static Module Bundler)입니다.

웹 애플리케이션은 여러 개의 JavaScript 파일, CSS 파일, 이미지 파일 등 다양한 종류의 파일들로 구성됩니다. 이러한 파일들은 각각의 의존성을 가지고 있고, 웹 페이지에서 로드되어야 합니다. Webpack은 이러한 파일들을 분석하여 의존성 관리를 수행하고, 필요한 파일들을 하나로 묶어서 번들 파일을 생성합니다.

Webpack은 모듈 시스템을 지원하는 JavaScript 파일을 처리할 수 있으며, CommonJS나 AMD 등과 같은 다양한 모듈 형식을 사용할 수 있습니다. 또한, 웹 애플리케이션의 성능을 향상시키기 위해 코드 압축, 리소스 최적화, 지연 로딩 등 다양한 기능을 제공합니다.

Webpack의 설정 파일을 사용하여 웹 애플리케이션의 빌드 프로세스를 커스터마이징할 수 있으며, 로더(Loader)와 플러그인(Plugin)을 활용하여 다양한 작업을 자동화할 수 있습니다. 예를 들어, Babel 로더를 사용하여 ES6+ 문법을 하위 버전의 JavaScript로 변환하거나, CSS 로더를 사용하여 CSS 파일을 번들에 포함시킬 수 있습니다.

Webpack은 현대적인 웹 개발에서 많이 사용되는 도구 중 하나로, 모듈화와 번들링을 통해 개발 생산성을 높이고 웹 애플리케이션의 성능을 최적화할 수 있는 강력한 기능을 제공합니다.

2.4 webpack.config.js

npx webpack을 작성하면 파일이 번들링 됩니다.

const path = require('path');

    module.exports = {
        mode: 'production',        //production, development
        entry: {
            index: './src/index.js'
        },
        output: {
            path: path.resolve(__dirname, 'public'),
            filename: 'js/[name].js'
        }
    }

2.8 CSS 파일 분리하기

npm install --save-dev mini-css-extract-plugin
const path = require('path');
    const HtmlWebpackPlugin = require('html-webpack-plugin');
    const MiniCssExtractPlugin = require("mini-css-extract-plugin");
    
    module.exports = {
        mode: 'production',        //production, development
        entry: {
            index: './src/index.js'
        },
        output: {
            path: path.resolve(__dirname, 'public'),
            filename: 'js/[name].js'
        },
        plugins: [
            new MiniCssExtractPlugin({
                filename: './css/style.css'     //번들 후
            }),
            new HtmlWebpackPlugin({
            template: './src/index.html',       //번들 전
            filename: './index.html',           //번들 후
            hash: true,
            showErrors: true
        })],
        module: {
            rules: [
              {
                test: /\.css$/i,
                // use: ["style-loader", "css-loader"],
                use: [MiniCssExtractPlugin.loader, "css-loader"],
                exclude: '/node_modules/'
              },
            ],
          },
    }
